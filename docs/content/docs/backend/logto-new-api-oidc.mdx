# Logto / New API 配置

本文以 `example.com` 作为统一域名示例，整理四部分配置：

- 可复制的 `.env` 模板
- Infinite Canvas 本地配置
- Logto 端配置
- New API 端需要设置的配置

## 1. 完整 `.env` 模板

```env
# 管理员账号，首次启动会自动创建。
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<admin-password>

# JWT 登录密钥和过期时间。
JWT_SECRET=<random-jwt-secret>
JWT_EXPIRE_HOURS=168

# Docker 镜像内后端监听 8080，前端监听 3000。
STORAGE_DRIVER=sqlite
DATABASE_DSN=/app/data/infinite-canvas.db

NEXT_PUBLIC_DOC_URL=https://docs.canvas.best
APP_PUBLIC_URL=https://canvas.example.com

LOGTO_ISSUER=https://auth.example.com/oidc
LOGTO_INTERNAL_ISSUER=
LOGTO_CLIENT_ID=<logto-client-id>
LOGTO_CLIENT_SECRET=<logto-client-secret>
LOGTO_SCOPE=openid profile email
SESSION_SECRET=<random-secret>
COOKIE_SECURE=true

NEW_API_BASE_URL=https://api.example.com
NEW_API_PUBLIC_URL=https://api.example.com
NEW_API_DISPLAY_NAME=New API
NEW_API_LOGTO_AUDIENCE=https://api.example.com
NEW_API_LOGTO_SCOPE=ecosystem:me ecosystem:models:read ecosystem:tokens:read
```

本地 Docker 调试时，通常只需要把公网地址改成本地端口：

```env
APP_PUBLIC_URL=http://localhost:3003
COOKIE_SECURE=false
NEW_API_BASE_URL=http://host.docker.internal:3000
NEW_API_PUBLIC_URL=http://localhost:3000
NEW_API_LOGTO_AUDIENCE=http://localhost:3000
```

`ADMIN_*`、`JWT_*`、`STORAGE_DRIVER` 和 `DATABASE_DSN` 来自当前根目录 `.env` 模板；当前 Next.js 主应用的 Logto 门禁主要读取 `APP_PUBLIC_URL`、`LOGTO_*`、`SESSION_SECRET`、`COOKIE_SECURE` 和 `NEW_API_*`。

## 2. Infinite Canvas 配置说明

说明：

- `NEXT_PUBLIC_DOC_URL` 是右上角文档入口地址。
- `APP_PUBLIC_URL` 是浏览器访问 Infinite Canvas 的地址。
- `LOGTO_ISSUER` 是 Logto 的 OIDC Issuer。
- `LOGTO_INTERNAL_ISSUER` 是服务端访问 Logto 的内部地址；不需要内网转发时留空。
- `LOGTO_CLIENT_ID` 和 `LOGTO_CLIENT_SECRET` 取自同一个 Logto 应用。
- `SESSION_SECRET` 用于加密 HttpOnly session cookie，必须设置足够长的随机字符串。
- `COOKIE_SECURE` 在 HTTPS 部署时设为 `true`，本地 HTTP 调试设为 `false` 或留空。
- `NEW_API_BASE_URL` 是 Infinite Canvas 服务端访问 New API 的地址。
- `NEW_API_PUBLIC_URL` 是浏览器打开 New API 配置页的地址，通常和 `NEW_API_BASE_URL` 相同。
- `NEW_API_DISPLAY_NAME` 只影响界面文案，不参与鉴权。
- `NEW_API_LOGTO_AUDIENCE` 是 Logto API Resource 的 audience。
- `NEW_API_LOGTO_SCOPE` 是 Infinite Canvas 需要向 Logto 申请的 New API 权限。

## 3. Logto 端配置

### 3.1 OIDC 应用

创建一个 OIDC 应用，填写回调地址：

```text
https://api.example.com/oauth/oidc
https://canvas.example.com/api/auth/logto/callback
```

登录 scope 使用：

```text
openid profile email
```

建议启用用户名补全，这样 Logto 会尽量返回 `preferred_username`。
如果没有补全，当前 New API 也会继续回退到 `username` 或 `email`。

### 3.2 API Resource

为 New API 创建一个 API Resource，示例 audience：

```text
https://api.example.com
```

为这个 Resource 添加权限：

```text
ecosystem:me
ecosystem:models:read
ecosystem:tokens:read
```

Infinite Canvas 会在登录时带上这些 scope，并请求对应 resource token。

## 4. New API 端需要设置的配置

### 4.1 OIDC 登录配置

在 New API 的 OIDC 设置里填写：

```env
oidc.enabled=true
oidc.client_id=<logto-client-id>
oidc.client_secret=<logto-client-secret>
oidc.well_known=https://auth.example.com/oidc/.well-known/openid-configuration
oidc.authorization_endpoint=https://auth.example.com/oidc/auth
oidc.token_endpoint=https://auth.example.com/oidc/token
oidc.user_info_endpoint=https://auth.example.com/oidc/me
ServerAddress=https://api.example.com
```

这一组配置用于 New API 自己的 OIDC 登录回调：

```text
https://api.example.com/oauth/oidc
```

### 4.2 Logto ecosystem 校验配置

如果要让 Infinite Canvas 登录后调用 New API 的 ecosystem 接口，还要设置：

```env
LOGTO_ECOSYSTEM_ENABLED=true
LOGTO_ISSUER=https://auth.example.com/oidc
LOGTO_AUDIENCE=https://api.example.com
LOGTO_JWKS_URI=https://auth.example.com/oidc/jwks
LOGTO_REQUIRED_SCOPES=
ECOSYSTEM_BASE_URL=https://api.example.com/v1
```

对应接口的 scope 为：

```text
ecosystem:me
ecosystem:models:read
ecosystem:tokens:read
```

## 5. 调用链路

Infinite Canvas 不会用固定管理员密钥读取 New API 数据。登录后，它会用当前 Logto 用户的 resource token 请求 New API：

```text
/api/new-api/config
-> /api/ecosystem/me
-> /api/ecosystem/models
-> /api/ecosystem/tokens
```

New API 根据 token 里的用户身份、audience 和 scope 返回当前用户可用的模型和令牌。

## 6. 字段对照

Logto `userinfo` 返回的字段里，New API 当前会按这个顺序取用户名：

1. `preferred_username`
2. `username`
3. `email`

因此用户补齐用户名后，最好让 Logto 返回 `preferred_username`；如果没有，New API 也能用邮箱兜底。

## 7. 快速检查

- `APP_PUBLIC_URL` 是否是浏览器真实访问 Infinite Canvas 的地址。
- Infinite Canvas 的回调地址是否落到 `https://canvas.example.com/api/auth/logto/callback`。
- 浏览器登录回调是否落到 `https://api.example.com/oauth/oidc`
- Infinite Canvas 是否能读到 `https://auth.example.com/oidc/.well-known/openid-configuration`
- New API 是否能访问 `https://auth.example.com/oidc/jwks`
- Logto API Resource 的 audience 是否和 `NEW_API_LOGTO_AUDIENCE`、`LOGTO_AUDIENCE` 一致
- 登录 token 的 scope 是否包含 `ecosystem:me ecosystem:models:read ecosystem:tokens:read`
